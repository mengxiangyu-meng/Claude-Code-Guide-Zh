## 1. 三种使用模式

Claude Code 提供三种使用模式，满足不同场景需求。

### 1.1 交互模式（Interactive Mode）

最常用的模式，启动一个持续的对话会话。

```bash
# 启动交互模式
claude

# 指定项目目录
claude /path/to/project
```

**特点：**
- 持续对话，Claude 会记住上下文
- 适合复杂任务、多轮交互
- 可以打断、修正、补充信息

**使用场景：**
- 代码审查
- 调试问题
- 架构设计

### 1.2 对话模式（Chat Mode）

单轮对话模式，适合快速问答。

```bash
# 单次对话
claude "如何用 Python 写一个快速排序？"

# 带文件参数
claude "这段代码有什么问题？" < file.py

# 多轮对话（带 --continue）
claude "写一个 hello world" --continue
```

**特点：**
- 每次都是新会话，无上下文
- 适合简单任务、快速查询
- 可以配合 shell 管道使用

**使用场景：**
- 语法查询
- 代码片段生成
- 概念解释

### 1.3 任务模式（Task Mode）

后台任务模式，适合长时间运行。

```bash
# 后台任务
claude --print "写一个 Python Web 服务器" > app.py

# 使用工具模式
claude --tool "Write" file_path="app.py" content="..."
```

**特点：**
- 不进入交互式对话
- 直接输出结果到 stdout
- 适合脚本集成

**使用场景：**
- CI/CD 流水线
- 自动化脚本
- 批量处理

### 模式对比

| 模式 | 命令 | 上下文 | 适用场景 |
|-----|------|-------|---------|
| 交互 | `claude` | 保持 | 复杂任务、调试 |
| 对话 | `claude "..."` | 一次性 | 简单问答、生成 |
| 任务 | `claude --print` | 无 | 脚本、自动化 |

## 2. 核心命令详解

### 2.1 基础命令

```bash
# 启动对话
claude

# 查看帮助
claude --help

# 查看版本
claude --version

# 检查配置
claude config show
```

### 2.2 对话控制命令

在交互模式中，可以直接使用以下命令：

| 命令 | 简写 | 说明 |
|-----|-----|------|
| `/quit` | `/q` | 退出对话 |
| `/resume` | `/r` | 恢复被中断的任务 |
| `/approve` | `/a` | 批准待处理的操作 |
| `/reject` | `/x` | 拒绝待处理的操作 |
| `/cost` | `/c` | 查看当前会话消耗 |
| `/clear` | | 清空对话历史 |

### 2.3 启动参数

```bash
# 指定模型
claude --model claude-opus-4-20250514

# 指定最大 Token 数
claude --max-tokens 4096

# 系统提示词
claude --system "你是一个 Python 专家"

# 危险模式（允许执行危险命令）
claude --dangerously-skip-permissions

# 工作树模式（隔离任务）
claude -w my-feature
```

### 2.4 模型选择

Claude Code 支持多种模型：

| 模型 | 适合场景 | 速度 | 智能 |
|-----|---------|-----|-----|
| `claude-opus-4-20250514` | 复杂推理、架构设计 | 慢 | 最高 |
| `claude-sonnet-4-20250514` | 日常编程（默认） | 中 | 高 |
| `claude-haiku-3-20250514` | 简单任务、快速问答 | 快 | 中 |

## 3. 项目上下文理解

Claude Code 强大的地方在于理解项目上下文。

### 3.1 自动上下文收集

当你启动 Claude Code 在一个项目目录时，它会自动：

1. **扫描项目结构** - 了解目录布局
2. **识别技术栈** - 检测使用的语言/框架
3. **读取配置文件** - package.json, requirements.txt 等
4. **解析依赖关系** - 理解模块间的关系

### 3.2 上下文范围

```bash
# 当前目录
claude

# 指定目录
claude /path/to/project

# 指定文件
claude README.md

# 指定多个文件
claude file1.py file2.js
```

### 3.3 上下文控制

```bash
# 限制上下文文件数
claude --max-turns 10

# 忽略某些文件
claude --no-read .git,node_modules

# 只读特定文件
claude --allowed-dir src
```

### 3.4 上下文提示词技巧

| 场景 | 提示词 | 效果 |
|-----|-------|------|
| 只改某部分 | "只修改 `auth.py` 中的登录逻辑" | 限定范围 |
| 不要改某部分 | "不要动 test 目录" | 排除范围 |
| 参考某风格 | "参考项目中的 utils.py 风格" | 风格一致 |

## 4. 会话管理

### 4.1 会话概念

每次启动 `claude` 都是一个新会话。会话信息保存在：

```bash
~/.claude/sessions/
```

### 4.2 常用会话操作

```bash
# 查看会话历史
claude sessions list

# 恢复会话
claude sessions resume <session-id>

# 删除会话
claude sessions delete <session-id>

# 导出会话
claude sessions export <session-id>
```

### 4.3 多会话并行

Claude Code 支持同时运行多个会话：

```bash
# 终端1：功能开发
claude -s feature-branch

# 终端2：bug 修复
claude -s bugfix-branch

# 每个会话独立运行，互不影响
```

### 4.4 会话续接

如果对话中断，可以使用 `/resume` 恢复：

```
> 我正在重构 auth 模块...
[中断]

# 重新连接后
> /resume
[继续之前的任务]
```

## 5. 权限模式

Claude Code 有多层权限控制，确保操作安全。

### 5.1 权限模式类型

| 模式 | 说明 | 适用场景 |
|-----|------|---------|
| `plan` | 只规划，不执行 | 复杂任务先看计划 |
| `BypassPermissions` | 完全信任 | 快速原型 |
| `AcceptEdits` | 接受所有编辑 | 日常开发 |
| `default` | 询问确认 | 保守模式 |

### 5.2 权限配置

```bash
# 查看当前权限
claude permissions show

# 设置权限模式
claude permissions set plan

# 临时提升权限（单次命令）
claude --dangerously-skip-permissions "rm -rf node_modules"
```

### 5.3 allowedTools 配置

可以精细控制允许使用的工具：

```json
// ~/.claude/settings.json
{
  "allowedTools": [
    "Read",
    "Write", 
    "Edit",
    "Bash",
    "grep"
  ]
}
```

## 6. 输出与日志

### 6.1 输出控制

```bash
# 只输出结果，不显示思考过程
claude --quiet "写一个函数"

# JSON 格式输出
claude --json "当前时间"

# 调试模式
claude --debug "你的请求"
```

### 6.2 日志管理

```bash
# 查看日志
claude logs

# 日志位置
~/.claude/logs/

# 日志轮转
# 自动保留最近 7 天的日志
```

### 6.3 Token 使用统计

```bash
# 查看当前会话消耗
/claude --cost
```

## 7. 常见问题

### Q1: 怎么退出交互模式？

```
# 使用 /quit 或 Ctrl+C
> /quit
再见！

# 或直接 Ctrl+C 两次
```

### Q2: 怎么让 Claude 不修改某些文件？

**方法一：提示词**
```
"不要修改 test/ 目录下的任何文件"
```

**方法二：配置排除**
```bash
claude --no-read test,dist,build
```

### Q3: 怎么重新开始一个会话？

```bash
# 方式一：直接启动新会话
claude

# 方式二：清空当前对话
/clear

# 方式三：指定新项目
claude /new/project/path
```

### Q4: 为什么 Claude 不执行命令？

检查权限设置：
```bash
claude permissions show
```

确保不是 `plan` 模式。

### Q5: 怎么查看 Token 消耗？

```
# 在交互模式中
/claude --cost
```

### Q6: 可以同时运行多个 Claude 吗？

可以，每个终端/标签页都是一个独立会话。
